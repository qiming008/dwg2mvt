# 高斯 - 克吕格投影配置说明

## 问题背景

DWG 图纸通常使用高斯 - 克吕格投影坐标系（Gauss-Krüger），而不是经纬度。例如：
- `gisPointX: 39620862.29` 
- `gisPointY: 4387832.11`

这是典型的高斯 - 克吕格 3 度带投影坐标，需要转换为 WGS84 经纬度才能在地图上正确显示。

## 坐标格式解析

高斯 - 克吕格坐标格式：`带号 + Y 坐标（含 500km 东偏移）+ X 坐标`

例如：`39620862`
- `39` = 投影带号（3 度带第 39 带）
- `620862` = Y 坐标（实际是 120862m，加上了 500km 的假东偏移）
- 中央经线 = 39 × 3 = 117°（适用于中国东部地区）

## 投影带号计算

### 3 度带（中国常用）
- **带号** = 经度 / 3（四舍五入）
- **中央经线** = 带号 × 3
- **适用范围**：中国大部分地区

| 带号 | 中央经线 | 适用地区 |
|------|----------|----------|
| 35 | 105° | 四川西部、青海 |
| 36 | 108° | 四川、重庆 |
| 37 | 111° | 湖北、陕西 |
| 38 | 114° | 河南、安徽 |
| **39** | **117°** | **北京、天津、山东、江苏** |
| 40 | 120° | 辽宁、吉林东部 |
| 41 | 123° | 吉林、黑龙江 |

### 6 度带
- **带号** = (经度 + 3) / 6（取整）
- **中央经线** = 带号 × 6 - 3

## 配置方法

### 方法 1：修改 .env 文件

在 `backend/.env` 文件中配置：

```env
# 启用高斯 - 克吕格转换
APP_ENABLE_GAUSS_KRUGER_TRANSFORM=true

# 设置投影带号（根据你的 DWG 图纸所在地区）
APP_GAUSS_KRUGER_ZONE=39

# 输出坐标系（WGS84 经纬度）
APP_TARGET_SRS=EPSG:4326
```

### 方法 2：根据图纸位置确定带号

1. **查看图纸上的坐标值**
   - 如果 X 坐标是 8 位数（如 39620862），前 2 位就是带号
   
2. **根据地理位置估算**
   - 北京/天津/山东：带号 39（中央经线 117°）
   - 上海/江苏/浙江：带号 39-40
   - 广东/广西：带号 37-38
   - 四川/重庆：带号 36

3. **使用在线工具查询**
   - 访问 [EPSG.io](https://epsg.io/)
   - 输入图纸所在位置的经纬度
   - 查询对应的 UTM/高斯 - 克吕格带号

## 验证转换结果

转换完成后，检查输出的坐标：
- **转换前**：X: 39620862, Y: 4387832（高斯 - 克吕格投影，单位：米）
- **转换后**：经度：117.x°, 纬度：36.x°（WGS84，单位：度）

如果转换后的经纬度在合理范围内（中国：经度 73-135°，纬度 3-54°），说明配置正确。

## 常见问题

### Q1: 转换后坐标不在中国范围内？
- 检查带号是否正确
- 确认是 3 度带还是 6 度带
- 尝试调整带号±1

### Q2: 不需要坐标转换？
- 如果 DWG 已经是经纬度坐标，设置 `APP_ENABLE_GAUSS_KRUGER_TRANSFORM=false`

### Q3: 如何知道图纸的准确带号？
- 查看图纸的坐标系说明
- 咨询设计单位
- 根据图纸位置反推

## 技术细节

### 使用的 PROJ.4 参数

```
+proj=tmerc +lat_0=0 +lon_0=117 +k=0.9996 +x_0=39500000 +y_0=0 +datum=WGS84 +units=m +no_defs
```

- `+proj=tmerc`：横轴墨卡托投影（高斯 - 克吕格）
- `+lat_0=0`：原点纬度（赤道）
- `+lon_0=117`：中央经线
- `+k=0.9996`：比例因子
- `+x_0=39500000`：东偏移（带号×1000000 + 500000）
- `+y_0=0`：北偏移
- `+datum=WGS84`：大地基准面
- `+units=m`：单位：米
